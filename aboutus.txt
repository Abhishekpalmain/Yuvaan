MISSION STATEMENT
================================================================================

To eradicate the corporate lending "Data Paradox" in India by building an
AI-powered, end-to-end automated Credit Decisioning Engine that reduces loan
appraisal turnaround time from 14 days to mere hours while maintaining full
regulatory compliance and explainability.

================================================================================
THE PROBLEM: THE DATA PARADOX
================================================================================

In Indian corporate lending, financial institutions face a "Data Paradox":
they have abundant data but cannot process it efficiently.

A typical mid-market loan application requires analyzing:
- Structured Data: Bank statements (years of data), GST filings (GSTR-1, 2A, 2B, 3B)
- Unstructured Data: 100+ page annual reports, board meeting minutes, legal notices
- External Intelligence: MCA compliance, e-Courts litigation, RBI macroeconomic data

Current process:
- Manual human review by credit analysts
- Takes 2-3 weeks per application
- Highly subjective (Credit Appraisal Memo - CAM)
- Prone to missing crucial early warning signals
- Results in increased NPAs and fraud losses

================================================================================
THE SOLUTION: THREE-PILLAR ARCHITECTURE
================================================================================

PILLAR 1: HIGH-LATENCY DATA INGESTOR
-------------------------------------
Specialized pipelines that parse and mathematically synthesize Indian financial data:

A. Intelligent Unstructured Parsing
   - Targeted OCR extraction for CIBIL Commercial Reports
   - Bounding-box localization to extract CIBIL Rank (1-10) and debt profiles
   - Handles inconsistent scanning and dense vernacular documents

B. Programmatic GST Reconciliation
   - Cross-references GSTR-3B (borrower's claimed ITC) vs GSTR-2A (supplier data)
   - Detects Input Tax Credit inflation/fraud
   - Mathematical synthesis (not just LLM reading)
   - Flags discrepancies: claimed ITC vs actual supplier filings

C. Graph Analytics: Circular Trading Detection
   - Builds directed graph from bank transactions (networkx)
   - Nodes: entities, Edges: fund transfers
   - Detects closed cyclical loops (e.g., Company -> ShellCorp A -> ShellCorp B -> Company)
   - Identifies artificial revenue inflation tactics

PILLAR 2: AUTONOMOUS RESEARCH AGENT
------------------------------------
Web-scale secondary research with forward-looking intelligence:

A. Web Scraping & Intelligence Gathering
   - Crawls MCA portal for company compliance status
   - Searches e-Courts for pending litigation
   - Scrapes financial news for sector-specific RBI regulations
   - Real-time macroeconomic headwind analysis

B. Primary Insight Integration Portal
   - UI for Credit Officers to input qualitative observations
   - NLP processing of human notes (e.g., "factory operating at 40% capacity")
   - Dynamically adjusts capacity scores based on real-world observations

PILLAR 3: RECOMMENDATION ENGINE
--------------------------------
Synthesizes all data into legally sound, transparent decisions:

A. Automated CAM Generation
   - Uses python-docx library to generate Word documents
   - Populates industry-standard "Five Cs of Credit":
     * Character: MCA/litigation scrape
     * Capacity: Cash flow analysis + GST reconciliation
     * Capital: Annual reports extraction
     * Collateral: Physical asset valuations
     * Conditions: Standard covenants

B. Explainable AI (XAI)
   - Simulates SHAP values for transparency
   - Every score deduction has explicit English explanation
   - Example output:
     * "-20 Points: GSTR-3B ITC claims exceed GSTR-2A supplier filings"
     * "-50 Points: Circular trading loop detected in Transaction TXN9902"
     * "-10 Points: Active high-severity litigation against promoters"
   - Regulatory compliant (no black-box AI)

   - GSTR_2A_B2B_Invoices: Array of supplier invoices with tax details
   - Shows intentional discrepancy for demo (claimed 300k IGST vs actual 414k)

3. RESEARCH REPORT (TechForge_Solutions_Pvt_Ltd_Research_Report.json)
   - Company, Sector_Risk, Litigation_Risk, Summary
   - Output from research_scraper.py

================================================================================
CODE MODULE DESCRIPTIONS
================================================================================

circular_trading.py (43 lines)
   - Reads bank_statement.json
   - Builds directed graph using networkx
   - Detects cycles using nx.simple_cycles()
   - Outputs detected loops or "No circular trading"

gstr_reconciliation.py (37 lines)
   - Reads gst_data.json
   - Compares GSTR-3B claimed ITC vs GSTR-2A actual supplier invoices
   - Calculates discrepancies for IGST, CGST, SGST
   - Returns list of warning messages

research_scraper.py (51 lines)
   - Simulates web scraping for MCA, e-Courts, RBI news
   - Generates company research report
   - Saves output as JSON to data/ folder

generate_cam.py (77 lines)
   - Accepts: company_name, base_score, gst_risk, circular_risk, cibil_rank
   - Applies explainable deduction rules
   - Generates recommendation: APPROVE / APPROVE WITH CONDITIONS / REJECT
   - Creates formatted Word document with Five Cs of Credit
   - Saves to data/ as CAM_<company>.docx

DEMO EXECUTION FLOW
================================================================================

1. Run gstr_reconciliation.py
   → Checks GST discrepancies

2. Run circular_trading.py
   → Detects circular trading loops in bank statements

3. Run research_scraper.py
   → Creates Research_Report.json with litigation/macro data

4. Run generate_cam.py
   → Combines all inputs, applies explainable AI
   → Generates final CAM document

- GST: Overclaimed IGST by INR 168,000 (300k claimed vs 414k actual)
- Circular Trading: Loop detected (3 entities found)
- Research: High litigation risk, Medium-High sector risk
